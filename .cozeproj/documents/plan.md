# 赋能方案自动推荐+推送联动改造

## 概述
改造赋能中心的核心逻辑：从"手动看方案列表"变为"系统根据新人不合格指标自动匹配推荐赋能方案→一键推送"。同步联动双轨诊断页、新人视图、带教视图、新人看板、总经理看板。

## 技术方案

| 维度 | 选择 | 理由 |
|------|------|------|
| 推荐引擎 | indicator_key 匹配 + 四象限补充 | 已有字段，无需新增表 |
| 处方结构 | 4段式(analysis/direction/prescription/standard) | 数据库已支持，需改前端渲染 |
| 联动入口 | 诊断页不合格指标→推荐→推送 | 核心业务流程 |
| 通知机制 | 推送后新人/带教同步可见 | 不新增通知表，用现有 empower_executions |

## 功能模块

### 核心改造：赋能方案自动推荐+推送

**现状问题**：
1. 赋能中心是独立页面，与诊断结果无联动
2. 方案卡片显示"关联指标X个"但不展示具体指标名
3. 方案详情展示逻辑有bug（content解析问题导致显示为空）
4. 推送方案时无处方预览

**改造目标**：
```
双轨诊断 → 发现新人X指标不合格
  ↓
系统自动匹配：indicator_key 关联 → 推荐"加V率提升方案"
  ↓
培训负责人确认推送 → 展示完整处方预览（病情分析+调理方向+药方+达标标准）
  ↓
推送后：
  - 新人：成长计划中看到完整处方+步骤可标记完成
  - 带教：看板中显示处方摘要+带教负责步骤+跟进点评入口
  - 诊断页：该新人标记"已推送赋能方案"
```

**推荐逻辑**：
- 逐项对标不合格指标 → 按 indicator_key 匹配 empower_plans
- 一个指标不合格 → 推荐1-2个方案
- 多指标不合格 → 按指标数量推荐，去重
- 四象限D类 → 额外推荐"综合提升方案"

**数据结构**（empower_plans.content 已支持）：
```json
{
  "analysis": "加V率低于60%，说明首通电话和回访环节未能有效传递价值",
  "direction": "强化首通电话价值传递能力，提升患者信任度",
  "prescription": [
    {"step": 1, "title": "首通电话话术复盘", "method": "带教老师陪同听录音+逐句分析", "duration": "3天", "responsible": "带教老师"},
    {"step": 2, "title": "话术模拟演练", "method": "按标准话术模板，模拟3种患者类型", "duration": "2天", "responsible": "新人"},
    {"step": 3, "title": "实战跟诊", "method": "带教老师示范→新人模仿→带教点评", "duration": "5天", "responsible": "带教老师+新人"}
  ],
  "standard": "连续2周加V率≥60%，首通电话质检评分≥4分"
}
```

### 联动模块改造

| 模块 | 改造内容 | 受益角色 |
|------|---------|---------|
| 双轨诊断页 | 不合格指标旁加"推荐方案"按钮→弹窗预览→一键推送 | 培训负责人 |
| 赋能中心 | 修复方案详情渲染bug + 卡片显示具体指标名(非数字) + 推送预览 | 培训负责人 |
| 新人视图 | 成长计划中展示完整处方+步骤可标记完成 | 新人 |
| 带教视图 | 看板中显示处方摘要+带教负责步骤+跟进点评 | 带教老师 |
| 新人看板 | 赋能超期/步骤逾期预警 | 培训负责人 |
| 总经理看板 | 赋能完成率细化(执行中/已完成/超期) | 总经理 |

6. **代码检查与验证** — test_run + push GitHub
