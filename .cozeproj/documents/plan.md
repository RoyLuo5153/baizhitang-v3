# 演练任务角色分化修复

## 概述
修复演练任务页面(practice)的角色视图混淆问题：新人应该接收和完成任务，而非审核或创建任务。带教老师/培训负责人审核任务完成情况。

## 技术方案

| 维度 | 选择 | 理由 |
|------|------|------|
| 框架 | 现有 Next.js + React | 仅修复前端逻辑，不改架构 |
| API | 现有 /api/practice + /api/practice/tasks | API已支持角色分化，前端未正确使用 |
| 数据库 | 无变更 | 后端逻辑已正确 |

## 功能模块

### 新人视角（需修复）
- 统计卡片：待完成 / 已提交 / 已审核
- Tab标签：待完成 | 我的提交
- 列表：显示分配给新人的待完成演练任务
- 操作：点击"提交录音"上传文件完成任务
- 隐藏：不显示"创建临时演练任务"按钮
- 隐藏：不显示"待我审核"相关内容

### 带教老师/培训负责人视角（当前正确，微调）
- 统计卡片：待审核 / 已审核 / 核心动作
- Tab标签：待审核 | 审核历史
- 列表：显示新人提交的待审核演练
- 操作：播放录音 + 5级评分 + 审核通过/不通过
- 显示"创建临时演练任务"按钮

## 页面规格

##### @page(/practice) 演练任务

**核心职责**：新人完成演练任务提交录音；带教老师/培训负责人审核评分

**新人视图**：
- 统计卡片：待完成数 / 已提交数 / 已审核数
- Tab：待完成 | 我的提交
- 待完成列表：任务名称 / 知识点标签 / 截止时间 / 提交按钮
- 我的提交列表：任务名称 / 提交时间 / 审核状态 / 得分

**审核人视图**（带教老师/培训负责人）：
- 统计卡片：待审核数 / 已审核数 / 核心动作数
- Tab：待审核 | 审核历史
- 右上角"创建临时演练任务"按钮
- 待审核列表：新人名 / 任务名称 / 提交时间 / 审核按钮
- 审核弹窗：播放录音 + 5级逐动作评分 + 卡点说明

**交互说明**

| 元素 | 动作 | 响应 | 传参 | 备注 |
|------|------|------|------|------|
| 提交录音按钮 | 点击 | 弹出提交弹窗 | taskId | 仅新人可见 |
| 提交弹窗确认 | 点击 | 提交录音，刷新列表 | fileUrl, description | — |
| 审核按钮 | 点击 | 弹出审核弹窗 | submissionId | 仅审核人可见 |
| 审核弹窗确认 | 点击 | 提交评分，触发联动 | score, comment, pass | 低分触发赋能 |
| 创建临时任务按钮 | 点击 | 弹出创建弹窗 | — | 仅审核人可见 |
